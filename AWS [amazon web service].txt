        - using nginx:- in our project using nginx for to host our project
           -[cmd]  sudo apt update  [it update our system]
           -[cmd] sudo apt install nginx [installing nginx]
           -[cmd]  sudo systemctl start nginx  [starting nginx]
           -[cmd] sudo systemctl enable nginx  [ enable nginx]
           - copy code from dist(build files) to /var/www/html
  
           -[cmd]  sudo scp -r dist/* /var/www/html
            
                   sudo:- root level permission
                   scp:- for coping
                   -r :- recersively means all folders
                   dist/* :- dist inside all files
                   /var/www/html :- from folder
 
           - enable port :80 of your instances [by default nginx is port 80. need enable in aws instance ]
                 - go to aws ec2 inside click instance inside again click security heading column 
                   after click security grops inside edit inbound rules after click add rule button
                   and set port range 80 and info 0.0.0.0/0






Backend-deploying:- 
       - ubuntu@ip-172-31-20-185:~ git clone path[project path]
       - ubuntu@ip-172-31-20-185:~/TimeSheet-backend$ npm install
       - ubuntu@ip-172-31-20-185:~/TimeSheet-backend$ npm run start

       - instead of npm run start we use "pm2" This package runs server in production level.
       - ubuntu@ip-172-31-20-185:~/TimeSheet-backend$ 
       - ubuntu@ip-172-31-20-185:~/TimeSheet-backend$ npm install pm2 -g
       - ubuntu@ip-172-31-20-185:~/TimeSheet-backend$ pm2 start npm -- start

       - ubuntu@ip-172-31-20-185:~/TimeSheet-backend$ pm2 logs  
            => if [pm2 start npm -- start] command not working we use [pm2 logs]
       
       -  pm2 flush npm, pm2 list, pm2 stop <name>npm , pm2 delete <name>

       - give to custom processor name by using:- pm2 start npm --name "backend" -- start
